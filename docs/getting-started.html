Tandem · PM Operating Kit

Getting started

Tandem is a Claude Code plugin plus a _00-Project-Management/ scaffold. Getting started means installing the plugin, dropping the scaffold into your project, wiring the pm:* scripts, and confirming the layout — after which the lifecycle skills are available as slash commands and the board renders to HTML.

Prerequisites

  • Node.js ≥ 18 — the pm:* scripts are dependency-free Node stdlib (no npm install

    • step for the tooling itself).

  • Claude Code with plugin support — the skills are model-invoked and also work as explicit

    • slash commands.

  • A Chrome / Chromium / Edge binary and Node ≥ 22 — *only* for the optional headless

    • dashboard smoke (pm:smoke). Everything else runs without a browser.

There is no database, server, framework, or build toolchain to install. The "application" is the Markdown folder and the scripts that lint and render it.

Install

  1. Install the plugin from your Claude Code marketplace, or point Claude Code at a local

    2. checkout for development (claude --plugin-dir ./<plugin-dir>).

  2. On first use the plugin drops the _00-Project-Management/ scaffold into your project

    3. root (if not already present), registers the lifecycle skills under the plugin's command namespace, enables the two hooks (lint-on-edit, dashboard-on-stop), and inserts a slim PM rules block into your root CLAUDE.md.

Wire and verify

Run these from your project root:

npm run pm:install   # merge the pm:* scripts into your package.json + pin the folder layout
npm run pm:doctor    # health check: PM folder present, scripts wired, layout resolved

pm:install is idempotent — re-running it is safe. pm:doctor exits non-zero only on core wiring problems (a missing pm:lint or pm:dash), so it is a reliable gate in CI.

First run

Prove the kit is working by rendering the board:

npm run pm:lint      # validate the frontmatter contract across every artefact
npm run pm:dash      # render the Command Center to 42-Monitor/DASHBOARD.html

Open 42-Monitor/DASHBOARD.html in a browser — that interactive board is the live view of the plan. From here, use the lifecycle skills in order: anchor a North Star and OKRs under 00-Strategy/, draft a PRD, draft an epic, split it into features, split those into stories (with paired testplans), refine to ready, then execute and close. See the Command lifecycle for the step-by-step order and Features for what each skill does.

Everyday commands

CommandWhat it does

npm run pm:dashRegenerate the interactive Command Center (DASHBOARD.html).

npm run pm:docsRender this documentation/ folder to HTML.
npm run pm:allpm:lint → pm:monitor → pm:dash → pm:docs in one pass.
npm run pm:doctorRe-check wiring and layout health.

Troubleshooting

  • **command not found / npm: not recognized when a pm:* script runs.** The Node install

    • directory may not be on the shell PATH (notably Git Bash and the default PowerShell session on Windows, which omit C:\Program Files\nodejs). Put the runtime on PATH before re-running — PowerShell: $env:PATH = 'C:\Program Files\nodejs;' + $env:PATH; bash: export PATH="/c/Program Files/nodejs:$PATH" — or invoke the absolute binary.

  • pm:smoke exits 2 ("no browser found" / "needs Node ≥ 22"). The headless smoke drives

    • an already-installed system Chrome/Chromium/Edge over the DevTools Protocol with zero npm dependencies. It needs a browser binary (override with SMOKE_CHROME=<path>) and Node ≥ 22. Exit 2 means *blocked*, not a render failure — which is why it is deliberately not chained into pm:all. Run it as npm run pm:dash && npm run pm:smoke.

  • pm:install refuses to write. A present-but-malformed .claude/settings.json or

    • .claude-pm-config.json is refused rather than silently overwritten, to preserve your permissions.deny baseline. Fix the JSON and re-run.
