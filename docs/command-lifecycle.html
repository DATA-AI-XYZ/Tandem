Command lifecycle

The kit's slash commands form one canonical chain that carries work from strategy all the way to a merged phase. Each command does one job, leaves the artefacts in a known state, and points at the next command in the chain. This page is the map; the single source of truth for the order lives in the core skill.

The chain

The lifecycle commands run in this order:

  1. draft-okrs — turn a North Star into quarterly objectives and key results.
  2. draft-prd — capture the requirement as a PRD under 20-Requirements/.
  3. draft-epic — open an EPIC-NN, linked to an OKR or PRD section (no strategy link, no

    4. epic).

  4. split-into-features — decompose the epic into FEAT-NN.M files.
  5. split-into-stories — decompose each feature into STORY-NN.M.PP files, writing the

    6. paired TESTPLAN-NN.M.PP in the same step.

  6. refine-backlog — run the Definition-of-Ready checklist and flip a story to ready.
  7. execution-strategist — group ready stories into batched "chats" with serial/parallel

    8. lanes (a dry-run plan; it changes no statuses).

  8. execute-batch — run a chat's stories in order (or execute-story for a single

    9. story), flipping ready → in-progress and implementing the acceptance criteria.

  9. run-testplan — run every test case's command verbatim, marking PASS/FAIL and

    10. auto-raising a BUG on any failure.

  10. close-out-story — run the Definition-of-Done gate and flip the story to done.
  11. close-phase — gate the whole phase, compile the retro, and merge the phase branch

    12. back to main. This step is terminal.

Gates between the steps

The chain is gated, not just sequential — each transition checks a contract before it proceeds:

  • Definition of Ready guards entry to in-progress: a missing testplan, an unrunnable

    • test case, or an unmet dependency stops the story at refine-backlog.

  • Definition of Done guards entry to done: every acceptance criterion ticked, the

    • testplan green, lint clean, and the AI-review field resolved.

  • Phase gates guard the merge: every story in the phase must be done, the board clean,

    • and the lint and smoke checks green before close-phase surfaces a merge command.

Cadence and utility commands — weekly-monitor, monthly-retro, reflect, session-start, critique, peer-review — sit outside the chain; they observe and maintain the board rather than advance a single story along it.
