# scaffold/

This folder is the **template** the `Tandem` plugin deploys. When the plugin is
installed into a project, the contents of `scaffold/_00-Project-Management/` are copied into that
project's root, giving it the full PM operating structure — standards, templates, lifecycle
prompts, and the generator scripts — ready to use.

## What gets copied at install

```
scaffold/_00-Project-Management/
├── CLAUDE.md                       PM folder rules
├── 00-Strategy/ 10-Inbox/ 11-Backlog/ 12-Active/ 13-Releases/ 14-Retros/   (seeded, mostly empty)
├── 20-Requirements/ 30-Epics/ 31-Features/ 32-Stories/ 33-Testplans/ 34-Bugs/
├── 40-Decisions/ 41-Reports/ 42-Monitor/MONITOR.md
├── 90-Standards/                   SOP.md · DAILY-WORKFLOW.md · CLAUDE-CODE-CONFIG.md · PROJECT-CONTEXT.md
├── 91-Templates/                   one template per artefact type
├── 92-Prompts/                     lifecycle prompts
└── 93-Scripts/                     pm:lint, pm:dash, and friends
```

Plus, at the project root:

- A slim `CLAUDE.md` block inserted under the `<!-- PM-KIT-BLOCK -->` marker (safe to re-apply). If
  no root `CLAUDE.md` exists, one is created from `91-Templates/ROOT-CLAUDE.template.md`.
- A `CODEBASE-MAP.md` generated by `npm run pm:map` (`93-Scripts/generate-codebase-map.js`) —
  top-level directories with a one-line purpose + owner.
